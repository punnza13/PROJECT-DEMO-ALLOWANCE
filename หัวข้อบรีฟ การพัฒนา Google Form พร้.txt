หัวข้อบรีฟ: การพัฒนา Google Form พร้อมระบบคำนวณเบี้ยเลี้ยงอัตโนมัติ (Backend Logic)

1. โครงสร้างการรับข้อมูล (Form Structure)
ข้อมูลพนักงาน: ใช้ระบบ Mockup รหัสพนักงานตามปีที่เข้า (เช่น 2024/001) 
ข้อมูลโครงการ: ต้องระบุ Project ID ทุกครั้งที่มีการเบิก 
การเลือกประเภทวัน (Type Selection): ต้องมีตัวเลือกให้แยกรายวัน ได้แก่:
Working Day: วันทำงานเต็มวัน 
Travel Day: วันเดินทาง (ได้รับเฉพาะ Travel Allowance) 
บทบาทพิเศษ: หากเลือกเป็น Main Speaker ให้ใช้ลอจิกการคำนวณแบบ Start-End (ตามเวลาปฏิบัติงานจริง) 

2. ตรรกะการคำนวณหลังบ้าน (Backend Logic - Google Sheets & Apps Script)ให้ Developer ใช้หลักการคำนวณดังนี้:
สมการหลักสำหรับเบี้ยเลี้ยง ($Total$):
	$$Total = (Count\_Working \times Rate\_W) + (Count\_Travel \times Rate\_T)$$
โดยที่ $Rate\_W$ คืออัตราทำงานเต็มวัน และ $Rate\_T$ คืออัตราวันเดินทาง 
การคำนวณวันทำงาน: ใช้ฟังก์ชัน NETWORKDAYS.INTL ร่วมกับพารามิเตอร์ Bitmask (เช่น "0000011") เพื่อหักวันหยุดเสาร์-อาทิตย์ออกอัตโนมัติ 
ระบบคำนวณวิทยากร (Main Speaker):
ใช้สูตร TOTALHOURS(End_Time - Start_Time) เพื่อหาจำนวนชั่วโมง 
นำจำนวนชั่วโมงไปคูณกับอัตราค่าสมนาคุณวิทยากร (เช่น 600 หรือ 1,200 บาท) ตามระดับพนักงาน 
ใช้เงื่อนไข Show_If ในฟอร์ม เพื่อแสดงช่องกรอกเวลาเฉพาะเมื่อระบุว่าเป็นวิทยากรเท่านั้น
3. ระบบวิเคราะห์และ AI (Analytics & Notification)
Real-time Dashboard: ข้อมูลจากการส่งฟอร์มต้องสรุปเข้า Dashboard เพื่อดูยอดรวม Manday 
Anomaly Detection (AI Logic): ระบบต้องวิเคราะห์ความถี่การทำงานรายเดือน (เช่น นาย A ลงงานกี่วันต่อเดือน) 
Alert System: หากพนักงานมี Manday สูงผิดปกติ ให้ระบบทำการแจ้งเตือน (Red Alert) ให้ HR ทราบทันที
4. งานดีไซน์และเทคนิค (UI & Export)
Theme สี: กำหนดให้ใช้สีหลักเป็น #ED4724 
การส่งออกข้อมูล: ระบบต้องสามารถ Generate Google Sheets หรือ Export CSV เพื่อให้ HR นำไปใช้งานต่อได้ทันที

เน้นย้ำว่าเราต้องการลดการคำนวณด้วยมือ (Manual) เพื่อป้องกันข้อผิดพลาดของ HR โดยให้ระบบใช้ Virtual Columns หรือ Script หลังบ้านจัดการคำนวณยอดสุทธิในใบเบิกเดียวตามสมการเชิงเส้นที่ให้ไปครับ


ฟังก์ชันการจัดการข้อมูล (Data Management: Import & Export)
นอกเหนือจากตรรกะการคำนวณหลังบ้านแล้ว ให้เพิ่มฟีเจอร์เพื่อให้ HR และผู้ดูแลระบบทำงานได้สะดวกขึ้นดังนี้:
1. การส่งออกข้อมูล (Export)รูปแบบไฟล์: ต้องสามารถ Export ข้อมูลสรุปการเบิกจ่ายออกมาเป็นไฟล์ CSV หรือ Excel (.xlsx) ได้ 
ความละเอียดของข้อมูล:
รายละเอียดรายโครงการพร้อม Project ID และรหัสพนักงาน 
ผลรวมการคำนวณตามประเภท Working Day และ Travel Day 
ข้อมูลเวลาเข้า-ออก และค่าสมนาคุณสำหรับ Main Speaker 
สรุปจำนวน Manday รายบุคคลเพื่อใช้ในการวิเคราะห์ความถี่การทำงาน 
การเชื่อมต่อ: ข้อมูลทั้งหมดที่บันทึกผ่าน Google Form ต้องสามารถ Generate ออกมาเป็น Google Sheets ได้โดยอัตโนมัติ 
2. การนำเข้าข้อมูล (Import)
Bulk Upload: ระบบควรมีฟังก์ชันให้ HR สามารถ Import ไฟล์ CSV หรือ Excel เข้าไปในระบบเพื่อ:
อัปเดตรายชื่อพนักงานและรหัสพนักงาน (Mockup ID) เข้าระบบคราวละมากๆ 
นำเข้าข้อมูลประวัติการเบิกจ่ายเก่าเพื่อนำมาใช้ในส่วนของ AI Logic สำหรับวิเคราะห์พฤติกรรมการทำงานย้อนหลัง (Work Frequency) 
Data Validation: เมื่อมีการ Import ข้อมูล ระบบต้องตรวจสอบความถูกต้องของรูปแบบข้อมูล (เช่น รูปแบบวันที่ หรือ Project ID) ก่อนนำไปคำนวณตามตรรกะหลังบ้านที่วางไว้ 
3. การแสดงผลและการประมวลผล (Dashboard & AI)Real-time Integration: ข้อมูลที่ Import เข้ามาใหม่ต้องถูกนำไปคำนวณและแสดงผลบน 
Real-time Dashboard ทันที 
Anomaly Detection: ข้อมูลที่นำเข้าต้องถูกตรวจสอบโดยตรรกะ AI หากพบว่ามีพนักงานที่มีการเบิก Manday สูงเกินเกณฑ์ ต้องแสดง Alert สีแดงบน Dashboard ให้ชัดเจน

นอกจากระบบคำนวณเบี้ยเลี้ยงแล้ว ผมต้องการให้เพิ่มปุ่ม Import/Export ที่รองรับไฟล์ CSV และ Excel เพื่อให้ HR สามารถดึงข้อมูลดิบไปทำบัญชีต่อได้ และสามารถอัปโหลดข้อมูลพนักงานหรือประวัติการทำงานย้อนหลังเข้าระบบเพื่อให้ AI นำไปวิเคราะห์ความถี่การลงหน้างานได้ด้วยครับ
